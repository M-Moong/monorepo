# 배포 안정성 체크리스트

> wedding 앱 (Next.js 16 + Supabase + Vercel) 기준. `dev → main` PR 머지 전 통과해야 하는 기준을 정리한 문서.

---

## 1. 고려해야 할 영역

### 1-1. 코드 품질

| 항목                 | 이유                                                    |
| -------------------- | ------------------------------------------------------- |
| TypeScript 오류 없음 | 런타임 타입 오류는 실 사용자에게 그대로 노출됨          |
| ESLint 오류 없음     | React Hooks 규칙 위반 등 논리 버그로 이어지는 패턴 방지 |
| Prettier 포맷 통일   | diff 노이즈 제거, 리뷰 가독성 확보                      |

### 1-2. 빌드

| 항목                  | 이유                                                             |
| --------------------- | ---------------------------------------------------------------- |
| `next build` 성공     | 빌드 실패 = 배포 불가. 정적 생성 시 환경 변수 누락도 여기서 터짐 |
| 번들 사이즈 이상 없음 | 모바일 청첩장은 초기 로드 속도가 핵심 UX                         |
| 환경 변수 설정 완료   | Supabase URL/Key 등 런타임 의존 값 누락 시 기능 전체 비정상      |

### 1-3. 기능 동작

| 항목                 | 이유                                                |
| -------------------- | --------------------------------------------------- |
| 방명록 읽기/쓰기     | DB 연결 유일한 서버 기능 — 깨지면 사용자 흔적 소실  |
| 이미지 로드          | 실사진 9장 누락 시 placeholder가 그대로 노출됨      |
| BGM 재생             | 모바일 자동재생 정책 우회 로직 정상 여부 확인       |
| 카카오맵 / 공유 버튼 | URL·API 키 미연결 상태로 배포 시 빈 기능으로 노출됨 |

### 1-4. 모바일/크로스 브라우저

| 항목                   | 이유                                                        |
| ---------------------- | ----------------------------------------------------------- |
| iOS Safari 렌더링      | 웨딩 청첩장 수신자 대부분이 iOS 사용자                      |
| 뷰포트 높이 (`100dvh`) | iOS Safari의 동적 주소창으로 인한 레이아웃 깨짐 방지        |
| 터치 이벤트 / 스크롤   | 갤러리 Lightbox, HUD 스크롤 연동이 터치에서 정상 작동하는지 |
| 폰트 로드 실패         | Cormorant Garamond / Space Grotesk CDN 장애 시 폴백 확인    |

### 1-5. 성능

| 항목                         | 이유                                                 |
| ---------------------------- | ---------------------------------------------------- |
| 이미지 최적화 (`next/image`) | 원본 이미지 그대로 서빙 시 로드 시간 급증            |
| LCP 3초 이하                 | Cover 섹션이 First Screen — 느리면 이탈              |
| 스크롤 애니메이션 60fps      | CSS 애니메이션이 메인 스레드 블로킹 없이 동작해야 함 |

### 1-6. 보안

| 항목                  | 이유                                                                 |
| --------------------- | -------------------------------------------------------------------- |
| Supabase RLS 정책     | 방명록 DB에 RLS 없으면 누구나 전체 데이터 열람/수정 가능             |
| 환경 변수 노출 안 됨  | `NEXT_PUBLIC_` 아닌 서버 전용 키가 클라이언트 번들에 포함되지 않는지 |
| 방명록 입력 길이 제한 | 무제한 입력 허용 시 DB 용량 낭비 및 레이아웃 파괴                    |

### 1-7. 인프라 / Vercel 설정

| 항목                           | 이유                                                        |
| ------------------------------ | ----------------------------------------------------------- |
| Vercel 프로젝트 환경 변수      | `.env.local`은 로컬 전용 — Vercel 대시보드에 별도 등록 필요 |
| `monorepo` root 기준 빌드 설정 | `Root Directory: apps/wedding` 또는 Turborepo preset 확인   |
| 커스텀 도메인 SSL              | HTTPS 미적용 시 iOS에서 혼합 콘텐츠 차단 가능               |

## 2. 배포 전 통과 기준 (Gate)

PR을 `main`에 머지하기 전 아래 항목을 **순서대로** 체크한다.

### Gate 1 — 정적 검사 (자동화 가능)

```bash
pnpm check-types          # TypeScript 오류 0개
pnpm lint                 # ESLint 오류 0개 (warning은 허용)
pnpm format:check         # Prettier 포맷 위반 0개
```

- **기준:** 위 세 명령이 모두 exit 0으로 종료

### Gate 2 — 빌드

```bash
pnpm --filter wedding build
```

- **기준:** 빌드 성공 + `Route (app)` 출력에 오류 페이지 없음
- **확인 포인트:** 환경 변수 missing warning이 콘솔에 없는지

### Gate 3 — 기능 수동 확인 (로컬 or Vercel Preview)

| #   | 확인 항목    | 판단 기준                                    |
| --- | ------------ | -------------------------------------------- |
| 1   | 방명록 작성  | 입력 후 새로고침해도 데이터 유지됨           |
| 2   | 방명록 읽기  | 기존 항목 정상 표시                          |
| 3   | 이미지 전체  | 9개 섹션 모두 실사진 표시, broken image 없음 |
| 4   | BGM          | 사용자 탭 후 재생 시작, 토글 정상            |
| 5   | 카카오맵     | 버튼 탭 시 지도 앱 또는 웹뷰로 이동          |
| 6   | 공유 버튼    | 카카오/링크/SMS 중 구현된 것 정상 동작       |
| 7   | HUD 스크롤   | 챕터 이동 시 위치 표시 정확                  |
| 8   | Fortune 카드 | 랜덤 명언이 매 탭마다 다르게 표시됨          |

### Gate 4 — 모바일 확인

- **기기:** 실제 iOS Safari (시뮬레이터 아닌 실기기 권장)
- **기준:**
  - 전체 섹션 스크롤 시 레이아웃 깨짐 없음
  - Lightbox 열기/닫기 터치 정상
  - 100dvh 영역 주소창에 가려지지 않음

### Gate 5 — Vercel Preview 배포 확인

- PR 생성 시 Vercel Preview URL 자동 생성
- **기준:** Preview URL에서 Gate 3 체크리스트 재확인
- **이유:** 로컬에서는 환경 변수가 있어도 Vercel에 미등록이면 기능 비정상

---

## 3. CI 자동화 권장 (미래)

현재 테스트 러너가 없으므로 Gate 1~2는 GitHub Actions로 자동화할 수 있다.

```yaml
# .github/workflows/ci.yml 예시
- run: pnpm check-types
- run: pnpm lint
- run: pnpm format:check
- run: pnpm --filter wedding build
```

Gate 3~5는 사람이 확인해야 하므로 PR 템플릿에 체크박스로 포함하는 방식이 현실적이다.

---

## 4. 배포 후 확인

| 항목                  | 확인 방법                                 |
| --------------------- | ----------------------------------------- |
| Vercel 빌드 로그 정상 | Vercel 대시보드 → Deployments → 최신 빌드 |
| 방명록 DB 연결        | 프로덕션 URL에서 방명록 작성 테스트       |
| 커스텀 도메인 HTTPS   | 브라우저 주소창 자물쇠 아이콘 확인        |
| Core Web Vitals       | Vercel Analytics 또는 PageSpeed Insights  |

---

> **원칙:** Gate를 통과하지 못하면 머지하지 않는다. 빠른 배포보다 손님이 보는 화면이 깨지지 않는 것이 우선이다.
